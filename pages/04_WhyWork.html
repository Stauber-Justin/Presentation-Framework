Warum funktioniert das überhaupt?

SQL-Injection entsteht, wenn Eingaben ungefiltert in SQL-Befehle eingebaut werden. Der Code denkt, die Eingabe ist ein Wert – aber sie ist in Wahrheit ein SQL-Befehl!

// ❌ Unsicheres Beispiel (gefährlich!)
const query = "SELECT * FROM users WHERE name = $input";
// Sicher mit Prepared Statement (parametrisiert)
const query = "SELECT * FROM users WHERE name = ?";
db.execute(query, [input]);
„Daten und Code sollten nie in einem String verheiratet werden.“
