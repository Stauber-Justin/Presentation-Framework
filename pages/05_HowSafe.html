Wie schützt man sich vor SQL-Injection?

  • ✔️ Prepared Statements (empfohlen!)
  • ✔️ Strikte Input-Validierung & Typprüfung
  • ✔️ Prinzip der minimalen Rechte (Least Privilege)
  • ✔️ Nutzung von Stored Procedures

Was sind Stored Procedures?

Stored Procedures sind vordefinierte SQL-Befehle, die direkt in der Datenbank gespeichert werden. Sie nehmen Parameter entgegen und trennen Logik von Eingaben – ganz ohne String-Konkatenation im Code! 


            CREATE PROCEDURE GetUserByName(@name NVARCHAR(50))
            AS
            BEGIN
              SELECT * FROM users WHERE name = @name;
            END

Vorteile: schwerer zu manipulieren, bessere Performance & Wartbarkeit.

„SQL-Injection ist kein Bug – sie ist eine Einladung.“
